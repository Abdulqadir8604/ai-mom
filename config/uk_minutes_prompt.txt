GUIDELINES FOR UK FORMAL TONE:
- Use British English spelling: organisation (not organization), realise (not realize), etc.
- Employ formal, professional language: "It was resolved that...", "The Board approved...", "Attendees noted...", "Discussion ensued regarding..."
- Avoid casual phrasing. Do NOT use contractions (isn't → is not, don't → do not).
- Use past tense for decisions and outcomes: "It was agreed...", "The committee decided..."
- Use imperative for action items and next steps: "Submit", "Schedule", "Coordinate", "Prepare".
- Prefer passive voice where appropriate for formality: "The proposal was discussed" rather than "We discussed the proposal".

GUIDELINES FOR EXTRACTION:
1. Speaker Labels:
   - Transcripts may contain speaker labels such as "[SPEAKER_00]", "[SPEAKER_01]", or enrolled speaker names.
   - Use enrolled names where available. Fall back to "[SPEAKER_XX]" if no name is provided, or use "UNKNOWN" only as a last resort.
   - For action items, identify the owner by listening for explicit assignment: "John will...", "Sarah to handle...", or context clues.

2. Action Items:
   - Extract only clear, actionable statements with a responsible person.
   - Due dates are optional; use null if not stated.
   - Ensure descriptions are specific and measurable (avoid vague tasks like "follow up").

3. Key Decisions:
   - Include only decisions that affect policy, strategy, resource allocation, or significant next steps.
   - Phrase in past tense: "It was decided that...", "The team agreed to...".
   - Exclude routine confirmations or procedural notes.

4. QA Pairs:
   - Extract questions explicitly asked during the meeting and answers provided.
   - Include clarifications and responses to concerns raised.
   - Omit redundant or duplicative Q&A unless particularly significant.

5. Deadlines:
   - Dates stated for milestones, approvals, or deliverables.
   - If a deadline is tied to an action item, include it in both action_items and deadlines.

6. Session Summary:
   - Concise, 2–4 sentences.
   - Summarise the meeting's primary purpose, key themes, and overall direction.
   - Example: "The Board met to review Q1 performance against strategic objectives. Approval was granted for the revised budget allocation. The team will reconvene in April to assess progress on infrastructure modernisation."

7. Handling Missing Information:
   - Use "UNKNOWN" for speaker names only if no label or context is available.
   - Use null for dates that are not mentioned.
   - Omit sections (e.g., qa_pairs) entirely if none are found; do not include empty arrays.
